# 并行启动指南

> 三人三机，从零到能并行开发的最短路径。按顺序做完，即可开工。

---

## 一、首次环境准备（每人各自做一次）

前置：已装 **Node.js ≥ 20.9**（推荐 22）、**Git**、**VSCode**（或你顺手的编辑器）。

```powershell
# 1. 克隆仓库（确认你有 collaborator 权限；没有让仓库 owner 邀请你）
git clone git@github.com:nuaaweixinye/A3-test.git
cd A3-test

# 2. 确认在 main 分支并是最新
git checkout main
git pull

# 3. 装依赖
npm install

# 4. 复制环境变量模板
cp .env.example .env
#   —— 暂时留空也能跑（mock 模式）；拿到星火密钥后填 SPARK_API_KEY

# 5. 启动开发服务器
npm run dev          # 浏览器开 http://localhost:3000
```

> 若提示 SSH 无权限：先配好 GitHub SSH key（`ssh-keygen` → 把 `~/.ssh/id_ed25519.pub` 加到 GitHub 账号 Settings → SSH keys）。

## 二、认领任务并开分支

任务清单见 `docs/任务看板.md`，规范见 `CONTRIBUTING.md`。

```powershell
# 每次开工前：同步主干
git checkout main
git pull --rebase

# 建你认领的任务分支
git checkout -b feature/你的任务名      # 例如 feature/spark-real-key

# 改代码……改完自检必过：
npx tsc --noEmit                        # 类型
npm run lint                            # 规范
npm run build                           # 构建（提交前必过）

# 提交
git add -A
git commit -m "feat: 简短中文描述"
git push -u origin feature/你的任务名

# 然后去 GitHub 网页点「Compare & pull request」，@队友 review
```

## 三、设置 main 分支保护（owner 做一次）

仓库 **Settings → Branches → Add branch rule**：
- Branch name pattern：`main`
- ✅ Require a pull request before merging → Require approvals：`1`
- ✅ Require status checks to pass（配好 CI 后再勾）
- 保存。之后任何人都不能直接推 `main`，必须走 PR。

## 四、每日同步与冲突速查

```powershell
# 每天第一件事：把主干最新合进自己分支
git fetch origin
git rebase origin/main

# 有冲突时：编辑器解决冲突文件 →
git add <冲突文件>
git rebase --continue

# 放弃 rebase 回到起点
git rebase --abort
```

冲突原则：**在自己分支上解决，绝不在 `main` 上解**。

## 五、同一台机器多人同时跑（若偶尔需要）

每人用各自端口，互不干扰：

```powershell
# 队友 A（默认 3000）
npm run dev

# 队友 B
$env:PORT="3001"; npm run dev

# 队友 C
$env:PORT="3002"; npm run dev
```

> 三台机器各自开发则无需改端口，默认 3000 即可。

## 六、讯飞星火密钥：申请与共享（🅰 主线任务 A1）

**申请路径**：
1. 访问 https://www.xfyun.cn/ → 注册/登录（建议实名）
2. 控制台 → 创建应用 → 选用「星火大模型」
3. 在「模型记忆」/「API Keys」处获取 **APIKey**（即接口密钥/APIPassword）
4. （可选）领取免费 token 额度

**配置到项目**：
```powershell
# 编辑你机器上的 .env
SPARK_API_KEY=你拿到的密钥
SPARK_BASE_URL=https://spark-api-open.xf-yun.com/v1
SPARK_DEFAULT_MODEL=4.0Ultra
```
填好后重启 `npm run dev`，系统自动从 mock 切换为真实生成。

**共享给队友**：
- 密钥**不要进 Git**（`.env` 已被忽略）
- 在群里/私下发给队友，各自填进自己机器的 `.env`
- 或队友各自申请一个（讯飞对个人有免费额度）

## 七、常用命令速查

| 命令 | 作用 |
| --- | --- |
| `npm run dev` | 开发服务器（热更新） |
| `npm run build` | 生产构建（产出 standalone） |
| `npm run lint` | ESLint 检查 |
| `npx tsc --noEmit` | 类型检查 |
| `node .next/standalone/server.js` | 跑 standalone 生产包（无 Docker） |
| `docker compose up --build` | Docker 一键起（需装 Docker） |
