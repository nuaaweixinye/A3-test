# 前后端接口文档

> 项目：基于大模型的个性化资源生成与学习多智能体系统（软件杯 A3）
> 版本 v1.0　·　对应代码：`app/api/{learn,tutor,eval}/route.ts`、`backend/types/index.ts`

---

## 0. 总览

| 接口 | 方法 | 用途 | 响应类型 |
|------|------|------|----------|
| `/api/learn` | POST | 学习闭环：画像→路径→6 资源并行生成 | `text/event-stream`（SSE） |
| `/api/tutor` | POST | 智能辅导多轮答疑（加分项④） | `text/event-stream`（SSE） |
| `/api/eval` | POST | 学习效果评估（加分项⑤） | `application/json` |

**通用约定：**
- 所有接口均为 `POST`，请求体 `Content-Type: application/json`。
- 两个 SSE 接口的 `runtime = "nodejs"`、`dynamic = "force-dynamic"`，流式输出。
- 错误统一格式：`{ "error": "错误描述" }`（非流式）或 SSE `error` 事件（流式）。

---

## 1. SSE 报文格式

两个流式接口均采用标准 SSE，每条事件由两行组成，以空行分隔：

```
event: <事件类型>
data: <JSON 字符串>

```

- `event` 字段即 `AgentEvent.type`（learn）或 `delta`/`done`/`error`（tutor）。
- `data` 为该事件的 JSON 序列化字符串。
- 响应头固定：
  ```
  Content-Type: text/event-stream; charset=utf-8
  Cache-Control: no-cache, no-transform
  Connection: keep-alive
  X-Accel-Buffering: no
  ```

**前端消费方式（参考）：** 用 `fetch` + `ReadableStream` 手动解析，或使用 EventSource；注意需逐块按 `\n\n` 切分事件。

## 2. `/api/learn` —— 学习多智能体闭环

> 对应代码：`app/api/learn/route.ts`、`backend/graph.ts`

### 2.1 请求

```
POST /api/learn
Content-Type: application/json

{ "message": "我想学数据结构里的数组" }
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `message` | string | 是 | 学生的学习诉求/发言，不可为空 |

**校验：** `message` 缺失或为空 → 返回 `400 { "error": "缺少 message 字段" }`。

### 2.2 响应（SSE 事件流）

后端先发送一条 `status` 事件表示闭环启动（含运行模式 mock/spark），随后依次推送画像、路径、资源相关事件，最后 `done`。

**事件序列（时间轴）：**

```
status           ×1   闭环启动
  └（graph: profile_builder 节点）
profile          ×1   画像构建完成
  └（graph: path_planner 节点）
path             ×1   路径规划完成
  └（graph: 6 个资源节点并行 fan-out）
resource_start   ×6   每类资源开始（doc/quiz/mindmap/video/code/reading）
resource_delta   ×N   逐 token 流式片段（6 个资源交错推送）
resource         ×6   每类资源最终结果（含 fact_check/sources）
done             ×1   全部结束
```

> 若任意环节抛异常，改为推送一条 `error` 事件并关闭流。

### 2.3 各事件 payload

#### `status` —— 进度状态
```json
{ "type": "status", "agent": "profile", "message": "正在通过对话构建 6 维度学习画像…" }
```
| 字段 | 类型 | 说明 |
|------|------|------|
| `agent` | string | 发出状态的 Agent 名：`system`/`profile`/`planner`/`doc`/`quiz`/`mindmap`/`video`/`code`/`reading` |
| `message` | string | 人类可读的进度文案 |

#### `profile` —— 学生画像（6 维度）
```json
{
  "type": "profile",
  "profile": {
    "knowledge_level": { "数组": 35, "链表": 20 },
    "cognitive_style": "visual",
    "error_patterns": ["边界条件判断错误"],
    "learning_goal": "exam",
    "learning_pace": "medium",
    "interests": ["算法竞赛"],
    "updated_at": "2026-07-04T10:00:00.000Z"
  }
}
```
| 字段 | 类型 | 说明 |
|------|------|------|
| `profile.knowledge_level` | `Record<string,number>` | 知识点→0~100 掌握度 |
| `profile.cognitive_style` | `"visual"\|"auditory"\|"reading"\|"kinesthetic"` | 认知风格 |
| `profile.error_patterns` | string[] | 易错点 |
| `profile.learning_goal` | `"exam"\|"project"\|"research"\|"interest"` | 学习目标 |
| `profile.learning_pace` | `"fast"\|"medium"\|"slow"` | 节奏 |
| `profile.interests` | string[] | 兴趣方向 |
| `profile.updated_at` | string? | ISO 时间戳 |

#### `path` —— 学习路径
```json
{
  "type": "path",
  "path": {
    "path_title": "数组入门路径",
    "estimated_time": "约 45 分钟",
    "steps": [
      {
        "step": 1,
        "title": "数组基本概念",
        "description": "理解连续存储与随机访问",
        "resource_tasks": [
          { "type": "doc", "topic": "数组", "reason": "基础薄弱" },
          { "type": "quiz", "topic": "数组下标访问" }
        ],
        "estimated_minutes": 15
      }
    ]
  }
}
```

#### `resource_start` —— 某资源开始生成（前端据此弹出卡片占位）
```json
{
  "type": "resource_start",
  "id": "abc123",
  "resType": "doc",
  "title": "数组",
  "topic": "数组"
}
```
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 本次资源的唯一 id，后续 `resource_delta`/`resource` 用它关联 |
| `resType` | `"doc"\|"quiz"\|"mindmap"\|"video"\|"code"\|"reading"` | 资源类型 |
| `title` / `topic` | string | 标题 / 主题 |

#### `resource_delta` —— 流式片段（逐 token，多资源交错）
```json
{ "type": "resource_delta", "id": "abc123", "text": "# 数组\n" }
```
前端按 `id` 把 `text` 追加到对应卡片内容区，实现打字机效果。

#### `resource` —— 单个资源最终结果
```json
{
  "type": "resource",
  "resource": {
    "id": "abc123",
    "type": "doc",
    "title": "数组",
    "topic": "数组",
    "content": "# 数组\n数组是由相同类型元素组成的有序集合…",
    "sources": ["01-数组", "08-复杂度分析"],
    "fact_check": { "score": 100, "flagged": [], "checked": 3 },
    "created_at": "2026-07-04T10:00:05.000Z"
  }
}
```
| 字段 | 类型 | 说明 |
|------|------|------|
| `resource.content` | string | Markdown 正文（完整） |
| `resource.sources` | string[] | 引用来源（防幻觉第 4 层） |
| `resource.fact_check.score` | number | 有知识库佐证的占比 0~100 |
| `resource.fact_check.flagged` | string[] | 未找到佐证的可疑声明 |
| `resource.fact_check.checked` | number | 可核声明总数 |

#### `done` / `error`
```json
{ "type": "done" }
{ "type": "error", "message": "星火接口超时" }
```

### 2.4 前端状态映射（参考 `frontend/lib/store/useLearningStore.ts`）

| 收到事件 | 前端动作 |
|----------|----------|
| `status` | `setStatus(...)` 显示进度条/提示 |
| `profile` | `setProfile(...)` → `<ProfileRadar/>` 画雷达图 |
| `path` | `setPath(...)` → 渲染学习路径 |
| `resource_start` | `onResourceStart(...)` 新建卡片占位 |
| `resource_delta` | `onResourceDelta(id,text)` 追加内容 |
| `resource` | `upsertResource(...)` 标记完成（含 fact_check 徽标） |
| `done` | `setRunning(false)` 结束 loading |

## 3. `/api/tutor` —— 智能辅导（加分项④）

> 对应代码：`app/api/tutor/route.ts`、`backend/agents/tutor-agent.ts`

### 3.1 请求

```
POST /api/tutor
Content-Type: application/json

{
  "message": "数组的随机访问为什么是 O(1)？",
  "history": [
    { "role": "user", "content": "什么是数组？" },
    { "role": "assistant", "content": "数组是连续存储的…" }
  ],
  "profile": { /* StudentProfile，可省略 */ }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `message` | string | 是 | 本次问题，不可为空 |
| `history` | `TutorTurn[]` | 否 | 历史多轮对话（`role`∈`user\|assistant`） |
| `profile` | `StudentProfile\|null` | 否 | 当前画像，用于个性化答疑 |

**校验：** `message` 为空 → `400 { "error": "缺少 message 字段" }`。

### 3.2 响应（SSE 事件流）

事件类型仅三种：

```
delta   ×N   逐 token 文本片段
done    ×1   回答结束
error   ×1   （出错时替代 done）
```

#### `delta`
```json
{ "text": "因为数组在内存中连续存储，" }
```
> 注意：tutor 的 delta **没有** `type` 字段，前端直接拼 `data.text`。

#### `done` / `error`
```json
/* done 的 data 为空对象 */
{ }
/* error */
{ "message": "星火接口鉴权失败" }
```

## 4. `/api/eval` —— 学习效果评估（加分项⑤）

> 对应代码：`app/api/eval/route.ts`、`backend/agents/eval-agent.ts`

### 4.1 请求

```
POST /api/eval
Content-Type: application/json

{
  "profile": { /* StudentProfile */ },
  "progress": {
    "数组": { "mastery": 60, "viewed": true, "viewSeconds": 120 },
    "链表": { "mastery": 40, "viewed": true, "viewSeconds": 60 }
  },
  "topics": ["数组", "链表"]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `profile` | `StudentProfile\|null` | 否 | 当前画像 |
| `progress` | `Record<string,TopicProgress>` | 否 | 各主题的自评掌握度+浏览行为 |
| `topics` | string[] | **是** | 本次评估的主题列表（不可为空） |

**`TopicProgress` 结构：**
| 字段 | 类型 | 说明 |
|------|------|------|
| `mastery` | number | 学生自评掌握度 0~100 |
| `viewed` | boolean | 是否浏览过该资源 |
| `viewSeconds` | number | 浏览累计秒数 |

**校验：** `topics` 为空 → `400 { "error": "缺少 topics（请先生成学习资源）" }`。

### 4.2 响应（JSON）

成功 → `200`，返回 `EvaluationResult`：

```json
{
  "overall_score": 72,
  "mastery": { "数组": 70, "链表": 55 },
  "weak_points": ["链表"],
  "strong_points": ["数组"],
  "progress_trend": "improving",
  "recommendations": ["建议针对链表的指针操作再做练习"],
  "profile_update": { "knowledge_level": { "数组": 70, "链表": 55 } },
  "path_adjustment": {
    "action": "review",
    "focus_topics": ["链表"],
    "summary": "链表掌握度偏低，建议复习"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `overall_score` | number | 总体得分 |
| `mastery` | `Record<string,number>` | 主题→0~100 掌握度 |
| `weak_points` / `strong_points` | string[] | 薄弱 / 优势主题 |
| `progress_trend` | `"improving"\|"steady"\|"needs_review"` | 趋势 |
| `recommendations` | string[] | 学习建议 |
| `profile_update.knowledge_level` | `Record<string,number>` | **回写画像**（闭环：评估→画像） |
| `path_adjustment.action` | `"advance"\|"review"\|"steady"` | 路径调整动作 |
| `path_adjustment.focus_topics` | string[] | 重点复习主题（在 /learn 标徽标） |
| `path_adjustment.summary` | string | 调整说明 |

失败 → `500 { "error": "..." }`。

### 4.3 闭环用法

前端拿到 `EvaluationResult` 后：
1. `profile_update.knowledge_level` → 合并回 `StudentProfile`（`setProfile`），下次 `/learn` 带上更新后的画像。
2. `path_adjustment.focus_topics` → `setWeakTopics(...)`，在 `/learn` 资源卡片上标注「建议复习」。

## 6. 前端联调示例（基于 fetch + ReadableStream 解析 SSE）

```ts
// 消费 /api/learn（伪代码）
const resp = await fetch("/api/learn", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ message }),
});
const reader = resp.body!.getReader();
const decoder = new TextDecoder();
let buffer = "";
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });
  const parts = buffer.split("\n\n");
  buffer = parts.pop()!;          // 保留最后不完整的一块
  for (const part of parts) {
    const eventLine = part.match(/^event: (.+)$/m)?.[1];
    const dataLine = part.match(/^data: (.+)$/m)?.[1];
    if (!eventLine || !dataLine) continue;
    const evt = JSON.parse(dataLine);
    // 按 evt.type 分发到 store（见 2.4 表）
  }
}
```

> `/api/tutor` 解析方式相同，只是 `event` 只有 `delta`/`done`/`error`。
> `/api/eval` 为普通 JSON，直接 `await resp.json()` 即可。

## 7. 错误码汇总

| HTTP 状态 | 场景 | body |
|-----------|------|------|
| `400` | `/learn` 缺 message / `/tutor` 缺 message / `/eval` 缺 topics | `{ "error": "..." }` |
| `200` | SSE 正常开启（错误在流内以 `error` 事件返回） | event-stream |
| `500` | `/eval` 评估内部异常 | `{ "error": "..." }` |

> `/learn`、`/tutor` 的运行时错误**不**以 HTTP 5xx 返回，而是在已开启的 SSE 流中推送 `error` 事件后关闭流。
