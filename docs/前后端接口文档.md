# 前后端接口文档

> 项目：基于大模型的个性化资源生成与学习多智能体系统（软件杯 A3）
> 版本 v2.0

## 0. 接口总览

### 核心智能体接口
| 接口 | 方法 | 用途 | 响应类型 |
|------|------|------|----------|
| `/api/learn` | POST | 学习闭环：画像→路径→6资源并行生成 | SSE |
| `/api/tutor` | POST | 智能辅导多轮答疑 | SSE |
| `/api/eval` | POST | 学习效果评估（自动存入历史） | JSON |

### 认证接口
| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/auth/register` | POST | 注册（返回 Set-Cookie） |
| `/api/auth/login` | POST | 登录（返回 Set-Cookie） |
| `/api/auth/logout` | POST | 登出（清除 Cookie） |
| `/api/auth/me` | GET | 获取当前用户 |

### 数据接口
| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/profile` | GET | 获取已保存画像 |
| `/api/profile` | PUT | 保存画像 |
| `/api/profile/stats` | GET | 学习统计（会话/资源/评估/热力图） |
| `/api/profile/suggestions` | POST | AI 生成学习建议 |
| `/api/records` | GET | 学习记录列表 |
| `/api/records` | POST | 保存学习记录 |
| `/api/tutor/conversations` | GET | 辅导对话列表 |
| `/api/tutor/conversations` | POST | 新建对话+保存消息 |
| `/api/tutor/conversations/[id]` | GET | 单条对话消息 |
| `/api/eval/history` | GET | 评估历史列表 |
| `/api/kb/upload` | POST | 上传文档到 ChatDoc |
| `/api/kb/files` | GET | ChatDoc 文件列表 |

## 1. SSE 报文格式

```
event: <事件类型>
data: <JSON 字符串>

```

响应头：`Content-Type: text/event-stream; charset=utf-8`

前端通过 `frontend/lib/stream-manager.ts` 模块级消费 — 导航切换不中断。

## 2. `/api/learn` — 学习多智能体闭环

### 请求
```json
{ "message": "我想学Python编程" }
```

### 事件序列
```
status ×1        → 闭环启动
profile ×1       → 画像构建完成
path ×1          → 路径规划完成
resource_start ×6 → 各资源开始
resource_delta ×N → 逐 token 流式
resource ×6      → 最终结果（含 fact_check/sources）
done ×1          → 自动 saveRecord()
```

### 关键事件 payload

**profile**: `{ type, profile: StudentProfile }` — 6 维度画像

**resource_start**: `{ type, id, resType, title, topic }`

**resource_delta**: `{ type, id, text }` — 逐 token 追加

**resource**: `{ type, resource: { id, type, title, topic, content, sources, fact_check: { score, flagged, checked }, created_at } }`

## 3. `/api/tutor` — 智能辅导

### 请求
```json
{
  "message": "TCP三次握手是什么？",
  "history": [{ "role": "user", "content": "..." }, { "role": "assistant", "content": "..." }],
  "profile": { /* StudentProfile */ }
}
```

### 响应（SSE）
```
delta ×N   → { text: "逐token文本" }
done ×1    → 回答结束
```

问答完成后前端自动 `POST /api/tutor/conversations` 持久化。

## 4. `/api/eval` — 学习效果评估

### 请求
```json
{
  "profile": { /* StudentProfile */ },
  "progress": { "主题": { "mastery": 60, "viewed": true, "viewSeconds": 120 } },
  "topics": ["数组", "链表"]
}
```

### 响应（JSON · `EvaluationResult`）
```json
{
  "overall_score": 72,
  "mastery": { "主题": 70 },
  "weak_points": ["链表"],
  "strong_points": ["数组"],
  "progress_trend": "improving",
  "recommendations": ["..."],
  "profile_update": { "knowledge_level": { "主题": 70 } },
  "path_adjustment": { "action": "review", "focus_topics": ["链表"], "summary": "..." }
}
```

评估结果自动存入 `EvalRecord` 表。

## 5. 认证接口

### POST `/api/auth/register`
```json
// 请求
{ "username": "张三", "password": "123456" }
// 响应 200
{ "user": { "id": "...", "username": "张三" } }
// + Set-Cookie: auth_token=<jwt>; HttpOnly; Max-Age=604800
// 错误 400: 用户名长度/密码长度不足
// 错误 409: 用户名已存在
```

### POST `/api/auth/login`
```json
{ "username": "张三", "password": "123456" }
// 响应同 register，错误 401: 用户名或密码错误
```

### POST `/api/auth/logout`
```json
{ "ok": true }
// + Set-Cookie: auth_token=; Max-Age=0
```

### GET `/api/auth/me`
```json
{ "user": { "id": "...", "username": "张三" } }
// 未登录: 401 { "user": null }
```

## 11. 数据类型（`backend/types/index.ts`）

| 类型 | 说明 |
|------|------|
| `StudentProfile` | 6 维度学生画像 |
| `ResourceType` | doc/quiz/mindmap/video/code/reading |
| `LearningPath` | 学习路径（含步骤） |
| `GeneratedResource` | 生成资源（含 fact_check） |
| `ResourceCardState` | 前端卡片状态 |
| `TopicProgress` | 主题进度（掌握度+浏览） |
| `EvaluationResult` | 评估结果（含回写） |
| `AgentEvent` | SSE 事件联合类型 |
| `TutorTurn` | 辅导对话一轮 |

## 12. 错误码

| HTTP | 场景 | body |
|------|------|------|
| 400 | 参数缺失/格式错误 | `{ "error": "..." }` |
| 401 | 未登录（数据接口） | `{ "error": "未登录" }` |
| 409 | 用户名已存在 | `{ "error": "..." }` |
| 500 | 服务端异常 | `{ "error": "..." }` |
