# 系统开发说明书

> 项目：基于大模型的个性化资源生成与学习多智能体系统
> 赛题：第十五届中国软件杯 A3（出题企业：科大讯飞股份有限公司）
> 版本：v2.0　日期：2026-07

---

## 1. 概述

### 1.1 项目目标
面向高等教育个性化学习场景，借助大模型与多智能体协同框架，构建一套能够「对话式构建学生画像 → 规划个性化学习路径 → 并行生成多模态学习资源 → 智能辅导答疑 → 学习效果评估并回写画像」的完整闭环系统，切实落地"因材施教"。系统支持任意课程领域，不限定特定学科。

### 1.2 技术栈
| 层面 | 选型 |
| --- | --- |
| 全栈框架 | Next.js 16.2.10（App Router + Turbopack）+ React 19.2.4 + TypeScript 5 |
| 多智能体编排 | LangGraph.js 1.4.7（@langchain/core 1.2.1） |
| 大模型 | 科大讯飞星火（OpenAI 兼容端点 + WebSocket 流式） |
| 知识库 RAG | 讯飞星火知识库 ChatDoc（云端语义检索 + 事实核查） |
| 数据库 | Prisma 6 + SQLite（用户/画像/学习记录/辅导对话/评估历史） |
| 认证 | JWT（bcryptjs 密码哈希 + jose 签名）+ httpOnly Cookie |
| 前端 | Tailwind CSS v4、ECharts、react-markdown、Zustand、markmap-view |
| 流式通信 | Web ReadableStream + SSE（自定义事件协议） |
| 部署 | Docker（多阶段构建 + standalone 输出） |

### 1.3 相对 v1.0 的重大升级
1. **账号系统**：注册/登录/登出，JWT + SQLite 持久化，路由保护
2. **云端知识库**：从本地 md 文件切换为讯飞 ChatDoc 云端语义检索
3. **资源详情页**：6 种类型各有专用渲染器（交互答题、可视化导图、语法高亮等）
4. **学习画像仪表盘**：统计卡片 + 活动热力图 + AI 学习建议
5. **数据持久化**：画像/学习记录/辅导对话/评估历史全部落库
6. **通用化**：移除所有"数据结构与算法"硬编码，支持任意课程

## 4. 多智能体设计

### 4.1 编排图（LangGraph StateGraph）
```
START → profile_builder（画像构建Agent）
      → path_planner（路径规划Agent）
      ┌→ doc_gen       ┌→ quiz_gen      ┌→ mindmap_gen   （6 资源 Agent 并行）
      └→ video_gen      └→ code_gen      └→ reading_gen
      → resources 通道加法聚合 → END
```

### 4.2 八个角色 Agent
| Agent | 职责 |
| --- | --- |
| ProfileAgent | 对话抽取 6 维度画像 |
| PlannerAgent | 规划路径 + 输出资源任务清单 |
| Doc/Quiz/Mindmap/Video/Code/Reading Agent | 并行生成 6 类资源 |
| TutorAgent | 多轮答疑（RAG + 画像个性化），对话历史持久化 |
| EvalAgent | 学习效果评估 + 画像回写 + 路径调整建议，历史趋势追踪 |

## 5. 学习闭环（增强版）
```
画像 → 路径 → 6 资源生成 → 学习(测验得分回写 + 阅读标记 + 浏览采集)
        ↑                              ↓ 随时
        │                          ④智能辅导（持久化对话）
        │                              ↓
   画像回写 ← ⑤效果评估（雷达/弱点/建议/趋势图）→ 路径调整
```

### 5.1 新增闭环通路
- **测验 → 掌握度**：QuizDetail 提交后 `setMastery(topic, score)` 直接影响评估
- **阅读 → 浏览标记**：ReadingDetail 勾选已读 → `markViewed(topic)`
- **评估历史**：每次评估存入 EvalRecord，前端展示分数趋势折线图
- **画像编辑**：用户可手动调整知识点/兴趣/目标 → `PUT /api/profile`

---

## 6. 防幻觉机制（四层 · 云端升级）
1. **检索约束**：通过讯飞 ChatDoc 云端 API 语义检索（`backend/knowledge/spark-kb.ts`）
2. **Prompt 约束**：System Prompt 强制"仅基于知识库内容生成"
3. **事实核查**：ChatDoc 云端交叉验证，输出 score 与 flagged
4. **引用标注**：每条资源附 sources 来源

---

## 7. 资源详情页（6 种专用渲染器）
| 类型 | 组件 | 特性 |
| --- | --- | --- |
| 文档 | DocDetail | 目录侧栏 + scroll-spy 高亮 + 阅读进度条 |
| 题库 | QuizDetail | 交互答题 + 即时判分 + 错题解析 + 掌握度回写 |
| 导图 | MindmapDetail | markmap-view SVG 放射状图 + 缩放折叠 |
| 视频 | VideoDetail | SVG CSS 动画 + TTS 旁白 + DOMPurify 消毒 |
| 代码 | CodeDetail | react-syntax-highlighter 语法高亮 + 一键复制 |
| 阅读 | ReadingDetail | 外链卡片 + 已读勾选 + markViewed 回写 |

## 8. 账号与持久化
### 8.1 认证
- JWT（jose HS256）+ bcryptjs 密码哈希 + httpOnly Cookie
- `proxy.ts` 保护 `/learn` `/profile` `/tutor` `/eval` 路由
- NavShell 启动时自动从 DB 加载 profile + records

### 8.2 数据模型
| 模型 | 说明 |
| --- | --- |
| User | 用户账号 |
| Profile | 学生画像 6 维度 |
| LearnRecord | 学习会话（路径+资源+进度） |
| TutorConversation / TutorMessage | 辅导对话历史 |
| EvalRecord | 评估历史（分数+掌握度+弱点） |

### 8.3 持久化时机
| 事件 | 动作 |
| --- | --- |
| 画像更新 | 3s 防抖 PUT `/api/profile` |
| 学习完成 | POST `/api/records` |
| 辅导问答完成 | POST `/api/tutor/conversations` |
| 评估完成 | 自动存入 EvalRecord |
| 页面刷新 | NavShell 自动从 DB 恢复 |

## 9. 学习画像仪表盘
- **统计卡片**：学习会话数 / 资源数 / 辅导对话数 / 平均评估分
- **活动热力图**：90 天 GitHub 风格贡献日历 + 连续学习天数
- **评估趋势图**：ECharts 折线图展示历次评估分数变化
- **知识点分色**：< 40 红 / 40-70 黄 / > 70 绿
- **AI 学习建议**：调用星火 LLM 基于画像 + 评估生成个性化建议

## 10. 创新点
1. **云端语义 RAG**：讯飞 ChatDoc 替代本地关键词检索，召回更准
2. **完整数据闭环**：画像→资源→学习→评估→回写，全程持久化
3. **交互式资源**：测验判分、导图可视化、代码高亮 — 非纯文本
4. **学习分析仪表盘**：热力图 + 趋势图 + AI 建议 — 数据驱动
5. **导航安全流式**：stream-manager 模块级 SSE，切换页面不中断生成
6. **通用课程支持**：不限定学科，用户输入决定学习内容

## 11. 运行与部署
- 开发：`npm install && npx prisma db push && npm run dev`（http://localhost:3000）
- 生产：`npm run build`（产出 `.next/standalone`）
- Docker：`docker compose up --build`
- 一键启动：解压发行包 → 双击 `启动系统.bat`
- 接入星火：`.env` 填 `SPARK_API_KEY` + ChatDoc 凭证，未填则 mock 模式
