# 系统开发说明书

> 项目：基于大模型的个性化资源生成与学习多智能体系统
> 赛题：第十五届中国软件杯 A3（出题企业：科大讯飞股份有限公司）
> 版本：v1.0　日期：2026-07

---

## 1. 概述

### 1.1 项目目标
面向高等教育个性化学习场景，借助大模型与多智能体协同框架，构建一套能够「对话式构建学生画像 → 规划个性化学习路径 → 并行生成多模态学习资源 → 智能辅导答疑 → 学习效果评估并回写画像」的完整闭环系统，切实落地"因材施教"。

### 1.2 技术栈
| 层面 | 选型 |
| --- | --- |
| 全栈框架 | Next.js 16.2.10（App Router + Turbopack）+ React 19.2.4 + TypeScript 5 |
| 多智能体编排 | LangGraph.js 1.4.7（@langchain/core 1.2.1） |
| 大模型 | 科大讯飞星火（OpenAI 兼容端点 `spark-api-open.xf-yun.com/v1`） |
| 知识库 / 防幻觉 | 关键词重叠 RAG + 四层防幻觉（检索约束 / Prompt 约束 / 事实核查 / 引用标注） |
| 前端 | Tailwind CSS v4、ECharts、react-markdown、Zustand |
| 流式通信 | Web ReadableStream + SSE（自定义事件协议） |
| 部署 | Docker（多阶段构建 + standalone 输出） |

> 设计取舍：相对赛题开发指南的"Next.js + Python 双后端"方案，本项目简化为 **Next.js 全栈单语言**，降低部署与协作复杂度，多智能体能力由 LangGraph.js 在 Node 侧完整承担。

## 2. 需求分析

### 2.1 学生痛点
- 学习资源繁杂、难以精准匹配自身水平；
- 集体讲授无法兼顾个人节奏；
- 缺乏多模态、个性化的学习指导与即时答疑。

### 2.2 赛题功能映射
| 赛题要求 | 本系统实现 | 状态 |
| --- | --- | --- |
| ① 对话式学习画像构建（≥6 维度） | ProfileAgent，6 维度抽取 + 雷达可视化 | ✅ |
| ② 多智能体协同资源生成（≥5 种） | 6 类资源 Agent（doc/quiz/mindmap/video/code/reading）并行分发 | ✅ |
| ③ 个性化学习路径规划与推送 | PlannerAgent + 路径时间线 | ✅ |
| ④ 智能辅导（加分项） | TutorAgent 多轮答疑 + 朗读 | ✅ |
| ⑤ 学习效果评估（加分项） | EvalAgent + 雷达 + 画像回写 + 路径调整 | ✅ |
| 防幻觉与内容安全 | 四层防幻觉机制 | ✅ |

## 4. 多智能体设计

### 4.1 编排图（LangGraph StateGraph）
```
START → profile_builder（画像构建Agent）
      → path_planner（路径规划Agent）
      ┌→ doc_gen       ┌→ quiz_gen      ┌→ mindmap_gen   （6 资源 Agent 并行）
      └→ video_gen      └→ code_gen      └→ reading_gen
      → resources 通道加法聚合 → END
```
- 规划完成后**并行 fan-out** 到 6 个资源 Agent，产物经 `resources` 通道**加法 reducer** 聚合；
- 每个 Agent 经共享 `resource-runner` 统一完成「RAG 检索 → 星火流式 → 事件推送」；
- 节点通过 config 注入的 `emit` 回调，把 `resource_start / resource_delta / resource` 事件**逐 token 流式**推送至 SSE。

### 4.2 八个角色 Agent
| Agent | 职责 |
| --- | --- |
| ProfileAgent | 对话抽取 6 维度画像（知识基础/认知风格/易错点/目标/节奏/兴趣） |
| PlannerAgent | 规划路径 + 输出 ≥5 种资源任务清单 |
| Doc/Quiz/Mindmap/Video/Code/Reading Agent | 并行生成 6 类资源 |
| TutorAgent | 多轮答疑（RAG + 画像个性化） |
| EvalAgent | 学习效果评估 + 画像回写 + 路径调整建议 |

## 5. 学习闭环
```
画像 → 路径 → 6 资源生成 → 学习(自评掌握度 + 浏览行为采集)
        ↑                              ↓ 随时
        │                          ④智能辅导
        │                              ↓
   画像回写 ← ⑤效果评估（雷达/弱点/建议）→ 路径调整（/learn 弱点标注）
```
- 自评：`ResourceCard` 掌握度滑块写入 `progress`；
- 浏览：`IntersectionObserver` 累加可见时长；
- 回写：EvalAgent 输出 `profile_update.knowledge_level`，合并进画像；`path_adjustment.focus_topics` 在 `/learn` 标「建议复习」。

---

## 6. 防幻觉机制（四层）
1. **检索约束**：所有生成前先 RAG 检索知识库，作为 Prompt 上下文（`backend/knowledge/retriever.ts`）。
2. **Prompt 约束**：System Prompt 强制"仅基于知识库内容生成，不得编造，不确定须声明"。
3. **事实核查**：生成后 `factCheck` 抽取可核声明（复杂度/数值），回查知识库，输出 `score` 与 `flagged`（`backend/knowledge/fact-check.ts`），前端卡片展示「🔍 事实核查：N%」。
4. **引用标注**：每条资源附 `sources`（来源文件），卡片底部展示。

---

## 7. 创新点
1. **并行多智能体协同**：LangGraph fan-out 6 资源 Agent，加法聚合，体现"多智能体协同框架"。
2. **完整学习闭环**：画像→资源→学习→辅导→评估→回写，形成自适应迭代。
3. **可见的四层防幻觉**：含可量化的事实核查分数，区别于"仅 Prompt 约束"的常规方案。
4. **分阶段模型路由 + mock 兜底**：不同阶段用不同星火模型；无密钥自动 mock，保障可演示与可测试。
5. **单语言全栈 + 流式 SSE**：Next.js 全栈 + 原生 ReadableStream，部署简单、交互现代。

## 8. 运行与部署
- 开发：`npm install && npm run dev`（http://localhost:3000）
- 生产：`npm run build`（产出 `.next/standalone`）
- Docker：`docker compose up --build`（详见 docker-compose.yml）
- 接入星火：`.env` 填 `SPARK_API_KEY`（见 `.env.example`），未填则 mock 模式
