# 链表（Linked List）

链表是一种通过"指针（引用）"将零散内存节点串联起来的线性数据结构，节点在内存中不必连续。

## 分类
- **单链表**：每个节点包含 data 和指向下一节点的 next 指针，尾节点 next 为 null。
- **双链表**：节点同时持有 prev 与 next，可双向遍历。
- **循环链表**：尾节点指向头节点，形成环。

## 时间复杂度
| 操作 | 复杂度 |
| --- | --- |
| 按值查找 | O(n) |
| 访问第 i 个节点 | O(n) |
| 已知节点位置后插入/删除 | O(1) |

## 与数组对比
链表的优势在于插入/删除无需移动元素；劣势在于不支持随机访问，且每个节点需额外存储指针，空间开销更大。

## 常用技巧
- **虚拟头节点（dummy head）**：统一处理头部的插入/删除，避免特判。
- **快慢指针**：检测环、找中点。
- **反转链表**：迭代或递归。

## 易错点
1. 丢失指针导致节点"失联"——插入/删除时先用临时变量保存 next。
2. 忘记处理空链表或单节点链表的边界情况。
3. 反转链表时出现循环引用。
