# 查找算法（Searching）

查找是在数据集合中定位满足条件的元素。

## 顺序查找
逐个比较，时间 O(n)，适用于无序线性表。

## 二分查找（Binary Search）
要求序列**有序**。每次比较中间元素，将搜索区间减半。
- 时间复杂度 O(log n)，空间 O(1)。
- 关键：区间边界（左闭右闭 [l,r] 或左闭右开 [l,r)）与循环条件必须一致。

二分变体：
- 查找第一个 ≥ target 的位置（lower_bound）。
- 查找最后一个 ≤ target 的位置（upper_bound）。

## 哈希查找
通过哈希函数将 key 映射到桶，平均 O(1)。
- 冲突解决：链地址法（拉链）、开放寻址法。
- 负载因子过高时需扩容 rehash。

## 树形查找
BST / 平衡树 / B 树 / 红黑树，查找 O(log n)，适用于动态有序数据。

## 易错点
1. 二分查找死循环：区间定义与更新（l = mid + 1 还是 l = mid）不一致。
2. 哈希冲突处理不当导致查找效率退化。
3. 浮点数二分未控制好精度与迭代次数。
