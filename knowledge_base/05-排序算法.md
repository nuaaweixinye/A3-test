# 排序算法（Sorting）

排序将无序序列按规则（升序/降序）重新排列，是算法入门的核心。

## 常见排序对比
| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
| --- | --- | --- | --- | --- |
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 |

## 关键思想
- **分治**：归并排序（先递归拆分，再合并有序子序列）；快速排序（选基准 partition，左小右大再递归）。
- **堆**：堆排序利用完全二叉树的堆性质反复"取堆顶 + 下沉"。
- **稳定性**：相等元素排序后相对顺序不变即为稳定。

## 选择建议
小规模或基本有序用插入排序；大规模通用用快速排序；需要稳定且 O(n log n) 用归并；空间受限用堆排序。

## 易错点
1. 快排基准选择不当（如有序数组取首元素）退化为 O(n²)，可用随机化基准或三数取中优化。
2. 归并排序合并时辅助数组越界。
3. 混淆"稳定"与"时间复杂度"概念。
