# 算法复杂度分析（Complexity Analysis）

复杂度是衡量算法效率的工具，分为**时间复杂度**与**空间复杂度**。

## 渐进记号
- **O（大 O）**：上界，描述最坏情况下增长趋势，最常用。
- **Ω**：下界；**Θ**：确界。

## 常见时间复杂度（由小到大）
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

## 估算规则
- 单层循环遍历 n 次 → O(n)。
- 两层嵌套循环各 n 次 → O(n²)。
- 每次问题规模减半（如二分）→ O(log n)。
- 主定理（Master Theorem）分析分治递归 T(n) = aT(n/b) + O(nᵈ)。

## 空间复杂度
额外使用的内存：递归深度即栈空间；辅助数组、哈希表等。

## 常见误区
1. 只关注代码行数，忽略循环嵌套与递归规模。
2. 把 O(n log n) 与 O(n²) 混为一谈——n=10⁵ 时前者约 1.6×10⁶，后者约 10¹⁰，差距悬殊。
3. 忽略常数与低阶项在实际小规模数据上的影响（大 O 描述的是 n→∞ 的趋势）。

## 经验
- 1 秒内可执行的操作数约为 10⁷~10⁸，据此根据数据规模反推可接受的复杂度。
